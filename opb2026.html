PLAYBOOK · ONE PERSON BUSINESS 2026

กอบกู้ระบบบัญชีบริษัท ด้วย AI Agent ใน 6 ขั้น

คู่มือทำตามได้จริงทีละขั้น สำหรับเจ้าของธุรกิจ — มีระบบของตัวเองใน 1 วัน ทุกขั้นมี prompt ให้กดคัดลอกไปวางใน Claude Code ได้เลย

ใช้ Claude Codeไม่ต้องเขียนโค้ดภายใต้กฎหมายไทยโดย ชัยวัฒน์ ภูวนนท์จิรกวิน
ทางลัด 6 ขั้น
  1. รวมเอกสารทุกอย่างมากองรวมกัน
  2. สร้างกฎ + ทีม Agent 3 บทบาท
  3. วิเคราะห์ทั้งกอง หา Gap
  4. จัดโฟลเดอร์ให้เป็นระบบ
  5. งานประจำเดือน / ประจำปี
  6. ปฏิทินภาษี + การยื่นจริง
0

เตรียมเครื่องมือ (10 นาที)

เป้าหมาย: มี "สำนักงานบัญชี AI" หนึ่งโฟลเดอร์

1. ติดตั้ง Claude Code จาก claude.com/claude-code (แบบ Desktop app ง่ายสุดสำหรับคนไม่คุ้น Terminal)
2. สร้างโฟลเดอร์งาน เช่น บัญชี-บริษัทเรา/
3. เปิด Claude Code ชี้ที่โฟลเดอร์นี้ — จากนี้ทุกอย่างคือ "พิมพ์คุยกับ AI" ไม่ต้องเขียนโค้ดเองสักบรรทัด
1

รวมเอกสารทุกอย่างมากองรวมกัน

เป้าหมาย: เทของเข้า inbox เดียว — ไม่ต้องจัด ไม่ต้องเรียง

สร้างโฟลเดอร์ _กองเอกสาร/ แล้วเทไฟล์ทุกอย่างเท่าที่หาได้:

⭐⭐⭐
งบการเงินปีเก่า (ที่ผู้สอบบัญชีตรวจแล้ว) 2–3 ปี — ยอดปิดปีล่าสุด = "ยอดยกมา" จุดตั้งต้นของทุกอย่าง
⭐⭐⭐
แบบภาษีที่เคยยื่น + ใบเสร็จ (ภ.พ.30 · ภ.ง.ด.1/3/53 · ภ.ง.ด.50/51) — รู้ว่ายื่นถึงไหน ขาดตรงไหน
⭐⭐⭐
Statement ธนาคารทุกบัญชี ย้อนถึงต้นปีบัญชี — เงินจริงไม่โกหก ใช้ยันกับสมุดบัญชี
⭐⭐
หนังสือรับรองหัก ณ ที่จ่าย (50 ทวิ) · ใบกำกับภาษีขาย-ซื้อ
⭐⭐
หนังสือรับรองบริษัท (DBD) · บอจ.5 · ทะเบียน VAT (ภ.พ.20)
⭐
สัญญาเช่าซื้อ/เงินกู้ · ทะเบียนทรัพย์สิน + ตารางค่าเสื่อม
⚠️ อย่าลืม: บัญชีผู้ใช้ e-Filing สรรพากร + DBD — ถ้าผูกกับอีเมลนักบัญชีเดิม ให้รีบขอกู้สิทธิ์เข้าใช้เป็นอย่างแรก
2

สร้างกฎ + ทีม Agent 3 บทบาท

เป้าหมาย: วางกติกาครั้งเดียว ใช้ตลอดไป + มีทีมที่คานกันเอง

PROMPT 2.1 — สร้างกฎ (CLAUDE.md) 
ช่วยสร้างไฟล์ CLAUDE.md ให้คุณทำหน้าที่เป็น "นักบัญชีประจำบริษัท" ของผม โดยมีกฎเหล็ก:
1. ตัวเลขทุกตัวต้องอ่านจากเอกสารจริง (PDF/OCR) เท่านั้น ห้ามเดา ห้ามอ่านจากภาพด้วยสายตา
2. คำแนะนำภาษีทุกครั้งต้องอ้างอิงกฎหมายไทย (มาตราในประมวลรัษฎากร)
3. ก่อนแก้ไฟล์บัญชีใดๆ ต้อง backup ก่อนเสมอ และหลังแก้ต้องเปิดอ่านซ้ำ
   เพื่อยืนยันว่ารายการยังอยู่ครบ และเดบิต = เครดิต
4. บัญชีเงินฝากธนาคารในสมุดบัญชี ต้องเท่ากับ statement จริงเสมอ
   รายจ่ายที่ไม่ได้ออกจากบัญชีธนาคาร ห้ามลงว่าออกจากธนาคาร
5. ไม่แน่ใจให้บอกตรงๆ และแนะนำให้ถามผู้สอบบัญชี — ห้ามมั่ว
6. AI เตรียมงานได้ทุกอย่าง แต่การกดยื่นภาษีและการเซ็นเป็นหน้าที่มนุษย์
ใส่ข้อมูลบริษัทผมด้วย: [ชื่อบริษัท / เลขผู้เสียภาษี / รอบบัญชี / จด VAT หรือไม่ / ธนาคารที่ใช้]
PROMPT 2.2 — สร้าง Agent 3 บทบาท 
ช่วยสร้าง subagent 3 ตัวในโฟลเดอร์ .claude/agents/ ดังนี้

1) "นักบัญชี" — ทำงานบัญชีทั้งหมด: อ่านเอกสาร ลงสมุดรายวัน (Dr/Cr)
   คำนวณ VAT และภาษีหัก ณ ที่จ่าย ทำงบ ภายใต้กฎหมายไทยและกฎใน CLAUDE.md

2) "ผู้ตรวจสอบ" — นิสัยขี้สงสัย หน้าที่คือแย้งและจับผิดงานของนักบัญชี:
   ตัวเลขตรงเอกสารจริงไหม เดบิตเท่าเครดิตไหม ผิดกฎหมายข้อไหน มีความเสี่ยงอะไร
   ห้ามเกรงใจ ให้รายงานทุกจุดที่สงสัยแม้จะเล็กน้อย

3) "อาจารย์บัญชี" — รับงานที่นักบัญชีทำเสร็จแล้ว มาแปลเป็นภาษาคนธรรมดา
   สอนผมว่ารายการนี้คืออะไร ทำไมต้องทำ ครั้งหน้าผมทำเองยังไง ทีละขั้นแบบง่ายที่สุด
🔵 นักบัญชีลงมือทำ — ใต้กฎหมายไทย
🔴 ผู้ตรวจสอบแย้ง-จับผิด ก่อนถึงสรรพากร
🟢 อาจารย์บัญชีสอนเราให้เก่งขึ้นทุกเดือน
หัวใจคือการคานกัน: นักบัญชีทำ → ผู้ตรวจสอบแย้ง → งานผ่านการตรวจ 2 ชั้น ส่วนอาจารย์ทำให้เรา "เก่งขึ้นทุกเดือน" ไม่ใช่พึ่ง AI ตลอดชีวิต
3

วิเคราะห์ทั้งกอง หา Gap

เป้าหมาย: จากกองเอกสารมั่วๆ → แผนที่สถานการณ์ 1 ไฟล์

PROMPT 3 — วิเคราะห์ + หา Gap 
ให้ agent นักบัญชี อ่านเอกสารทุกไฟล์ใน _กองเอกสาร/ (PDF อ่านจาก text จริง,
รูปภาพให้ใช้ OCR) แล้วสรุปเป็นไฟล์ STATUS.md ตอบคำถามเหล่านี้:

1. บริษัทยื่นภาษีอะไรไปแล้วถึงเดือนไหน (ภ.พ.30 / ภ.ง.ด.1,3,53 / ภ.ง.ด.50,51)
2. อะไร "ค้าง" หรือ "ใกล้ครบกำหนด" บ้าง — เรียงตามความเร่งด่วน พร้อมวันตาย
3. งบปีล่าสุดปิดถึงปีไหน ยอดยกมาสำคัญมีอะไรบ้าง (เงินฝาก ลูกหนี้ เจ้าหนี้ เงินกู้กรรมการ)
4. เอกสารอะไร "หายไป" ที่ผมต้องตามหา เช่น statement เดือนไหนขาด บิลเดือนไหนไม่มี
5. ความเสี่ยงที่เห็นแล้วต้องบอกทันที (เช่น ค่าปรับ ยื่นล่าช้า ยอดไม่ตรง)

เสร็จแล้วให้ agent ผู้ตรวจสอบ ตรวจรายงานนี้ซ้ำ ว่าสรุปตรงกับเอกสารจริงไหม
ผลที่ได้: STATUS.md — รู้ทันทีว่าต้องทำอะไรก่อน-หลัง อะไรด่วน อะไรหาย
4

จัดโฟลเดอร์ให้เป็นระบบ

เป้าหมาย: ทุกไฟล์มีบ้าน — ของใหม่รู้ทันทีว่าเข้าช่องไหน

PROMPT 4 — จัดโครงสร้าง 00–06 
จัดไฟล์ทั้งหมดจาก _กองเอกสาร/ เข้าโครงสร้างโฟลเดอร์มาตรฐานนี้
(ย้ายจริง + เปลี่ยนชื่อไฟล์ให้สื่อความหมาย):

00_ข้อมูลบริษัท/       หนังสือรับรอง ทะเบียน VAT บอจ.5
01_งบการเงิน/          งบที่ตรวจแล้วแยกรายปี
02_ภาษีที่ยื่นแล้ว/     แยกชนิดแบบ → แยกปี → เก็บแบบ+ใบเสร็จคู่กัน
03_รายได้-ใบกำกับขาย/   ใบกำกับขาย + 50ทวิ ที่ถูกหัก
04_ภาษีซื้อ-รายจ่าย/    ใบกำกับซื้อแยกรายเดือน
05_ทรัพย์สิน-ค่าเสื่อม/  ทะเบียนทรัพย์สิน ตารางค่าเสื่อม
06_ระบบบัญชี/          สมุดบัญชี (Excel) + statement ธนาคาร + SOP

แล้วเขียน README.md อธิบายว่า "เอกสารใหม่แต่ละชนิด เข้าโฟลเดอร์ไหน"
เพื่อที่เดือนหน้าผมได้เอกสารมา จะรู้ทันทีว่าวางตรงไหน
กติกาถาวร: _กองเอกสาร/ = inbox เทของใหม่เข้าไปก่อนได้เสมอ แล้วสั่ง AI "จัดเข้าระบบ" — เราไม่ต้องจำอะไรเลย
5

งานประจำเดือน / ประจำปี

เป้าหมาย: เดือนแรกสร้าง SOP → เดือนต่อไปสั่งประโยคเดียว

PROMPT 5.1 — เดือนแรก: สร้าง SOP + สมุดบัญชี 
สร้างไฟล์ SOP_ปิดเดือน.md เป็นขั้นตอนปิดบัญชีรายเดือนของบริษัทผม:
รวบเอกสาร → อ่านตัวเลขจริง → ลงสมุดรายวัน → คำนวณภาษี → กระทบยอดธนาคาร → สรุปให้ผมยื่น
พร้อมสร้างสมุดบัญชี Excel: ผังบัญชี + ยอดยกมา (จากงบปีล่าสุดที่ตรวจแล้ว) + สมุดรายวัน
PROMPT 5.2 — ทุกเดือน: ประโยคเดียวจบ 
ปิดเดือน [มิถุนายน] ตาม SOP: ผมวางเอกสารใหม่ไว้ใน _กองเอกสาร/ แล้ว
(บิลซื้อ-ขาย, 50ทวิ, statement) ให้นักบัญชีจัดเข้าโฟลเดอร์ ลงบันทึกรายการ (JE)
คำนวณ ภ.พ.30 และภาษีหัก ณ ที่จ่าย กระทบยอดธนาคารให้ตรง statement
เสร็จแล้วให้ผู้ตรวจสอบตรวจซ้ำ แล้วให้อาจารย์สรุปสอนผม 5 บรรทัดว่าเดือนนี้เกิดอะไรขึ้น
PROMPT 5.3 — ประจำปี (สั่งช่วง ม.ค.–พ.ค.) 
เตรียมปิดงบประจำปี [2569]: สรุปงบทดลองทั้งปี ตรวจว่าทุกเดือนกระทบยอดแล้ว
รวมประเด็นที่ต้องให้ผู้ทำบัญชี/ผู้สอบบัญชี (CPA) ตัดสิน เป็น checklist พร้อมเอกสารแนบ
เพื่อส่งต่อสำนักงานบัญชีให้ปิดงบ + ยื่น ภ.ง.ด.50 + ส่งงบ DBD ได้ง่ายที่สุด
⚠️ กฎหมายไทย: งบประจำปีต้องมี "ผู้ทำบัญชี" (ขึ้นทะเบียน) + "ผู้สอบบัญชี (CPA)" — AI แทนสองตำแหน่งนี้ไม่ได้ แต่เตรียมงานให้จนเขาแค่ตรวจ+เซ็น
6

ปฏิทินภาษี + การยื่นจริง

เป้าหมาย: ไม่มีลืมยื่น — และมนุษย์เป็นคนกดเสมอ

PROMPT 6.1 — สร้างปฏิทินครั้งเดียว 
เพิ่ม "ปฏิทินภาษี" ของบริษัทผมลงใน CLAUDE.md:
- ภ.ง.ด.1/3/53 (หัก ณ ที่จ่าย) → ยื่น e-Filing ภายในวันที่ 15 ของเดือนถัดไป
- ภ.พ.30 (VAT) → ภายในวันที่ 23 ของเดือนถัดไป
- ภ.ง.ด.51 (ครึ่งปี) → 31 ส.ค.  |  ภ.ง.ด.50 (ทั้งปี) → ภายใน 150 วันหลังสิ้นรอบ
- ประชุมผู้ถือหุ้น (AGM) → ภายใน 4 เดือน  |  ส่งงบ DBD → ภายใน 1 เดือนหลัง AGM
และกำหนดว่า: ทุกครั้งที่ผมเปิดคุยเรื่องบัญชี ให้เช็คก่อนว่ามีอะไรใกล้ครบกำหนด
ใน 14 วัน แล้วเตือนผมเป็นอย่างแรก
PROMPT 6.2 — ต้นเดือน: ถามประโยคเดียว 
เดือนนี้ผมต้องยื่นแบบอะไรบ้าง ภายในวันไหน ตัวเลขแต่ละแบบเท่าไร
เตรียมสรุปตัวเลขให้ผมกรอก e-Filing ทีละช่องเลย
จบ loop รายเดือน: AI สรุปตัวเลขทีละช่อง → เราเปิด efiling.rd.go.th กดยื่นเอง จ่ายเอง → โหลดใบเสร็จวางใน _กองเอกสาร/ → สั่ง "เก็บใบเสร็จเข้าระบบ + ลงบันทึกจ่ายภาษี" ✅
!

กฎเหล็ก 5 ข้อ — ใส่ไว้ในระบบตั้งแต่วันแรก

ตัวเลขต้องมาจากเอกสารจริง — สั่งให้ AI อ่านจาก PDF text หรือ OCR เท่านั้น ไม่อ่านตัวเลขจากภาพด้วยสายตา
บัญชีธนาคารในสมุด = statement เสมอ — รายจ่ายที่จ่ายผ่านบัตรเครดิตหรือกรรมการ ห้ามลงว่าออกจากบัญชีธนาคาร
Backup ก่อนแก้ + เปิดอ่านซ้ำหลังแก้ — ทำกับไฟล์บัญชีสำคัญทุกครั้ง และยืนยันว่าเดบิต = เครดิต
ยอดต้องกระทบครบถึงสตางค์ — ผลต่างที่อธิบายไม่ได้ = ระเบิดเวลา พักไว้ใน "บัญชีพักรอผู้สอบบัญชี" ห้ามมั่ว
AI ไม่เซ็น ไม่กดยื่นแทน — ผู้ทำบัญชี + CPA + เจ้าของกดยื่นเอง คือ 3 มนุษย์ที่ระบบนี้ยังต้องมี
AI ทำงานเร็วได้ แต่ ความรับผิดชอบ ต้องมีคนคุม
