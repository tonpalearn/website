ภาพรวมโครงการ & วิสัยทัศน์

เปลี่ยน PMS จาก Excel 300+ ไฟล์ (350 คน · ~2 เดือน) → Single Source of Truth ดิจิทัลที่เชื่อม KPI · Project · Skill เข้าด้วยกัน — เห็นความเชื่อมโยง "ศักยภาพคน ↔ ผลลัพธ์ธุรกิจ" ไม่ใช่แค่เก็บคะแนน

🎯 Pain Point
Excel ช้า · ข้อมูลไม่ตรง 4 ระบบ (email mismatch) · หายตอน rotate · หัวหน้ามือแข็ง/อ่อน · พนักงานไม่ยอมกรอก
💡 Solution
Permanent Emp ID เป็นแกน · Survey-style token · Skill Matrix วัดได้จริง · Calibration + Defend
⚙️ Stack / โมเดล
GitHub · Vercel · Supabase · SaaS (เราดูแล code · ข้อมูล = กรรมสิทธิ์องค์กร)
กลยุทธ์เฟส: Phase 1 = Quick Win ขึ้นเร็วทันรอบประเมิน (เลิก Excel) · Phase 2 = Full Strategic Integration (PXP SSO + API + Skill Score ON) — ข้อเสนอนี้ตีราคา Phase 1 ชัดเจน, Phase 2 เป็น indicative
⚙️ Configurable by Design — ระบบเดียว ปรับให้ตรงแต่ละองค์กรได้
ไม่ hard-code — ผู้บริหาร/OD ตั้งค่าเองได้ 4 จุดสำคัญ (จุดขายที่ทำให้ระบบใช้ซ้ำได้หลายบริษัท)

Requirement Features

ฟีเจอร์หลักแยกตามโมดูล — P1 = Phase 1 (ในราคา) · P2 = Phase 2 · ⚙ CONFIG = ตั้งค่าได้

🔑 Access & Identity

  • Token Link รายบุคคล (signed, หมดอายุ) — Survey-style P1
  • Permanent Employee ID เป็น PK (email เปลี่ยนได้ ประวัติไม่หาย) P1
  • RBAC 5 ระดับ (Admin/VP/Manager L1/Reviewer L2/Employee) P1
  • PXP SSO param gateway P2

🗂 Master Data + KPI Setup

  • KPI หลากรูปแบบต่อแผนก ⚙ CONFIG P1
    รอบวัด: เดือน/ไตรมาส/ครึ่งปี/ปี · หน่วย: ฿ / จำนวนชิ้น / % / count · calc: ACC/AVE — เช่น Sales รายเดือน(฿), ฝ่ายผลิตรายไตรมาส(ชิ้น)
  • Weighting Matrix ราย 4 ระดับ ⚙ CONFIG P1
  • Corporate KPI + 15 dept KPI libraries + Difficulty (×1.0/1.3/1.5) P1

📌 Project-based

  • Cascade จากองค์กร ⚙ CONFIG P1
    องค์กรตั้ง Strategic ปีนี้ (เช่น Kaizen, Cost-reduction) → พนักงานกรอกว่าทำอะไร อยู่ใต้โครงการไหน → เห็นภาพรวมทั้งองค์กร
  • Strategic Multiplier ×1.5 (โครงการสำคัญ) + แนบ evidence · คะแนน shared P1
  • PM Pro (task/%/dependency) P2

🌱 Skill / Competency

  • เลือกโหมดวัดได้ ⚙ CONFIG P1
    A) Skill Score (Skill Domain Matrix · proficiency 1–5) · B) Competency — มาตรฐานสากล (4 Corporate/ASK) หรือ ผู้บริหารระบุเอง (custom)
  • Gap Analysis + Skill Radar · baseline น้ำหนัก 0% (Phase 1) P1
  • Skill Score ON 15–20% + LMS feed P2

📊 KPI Evaluation

  • กรอก actual ตามรอบที่ตั้ง (เดือน/ไตรมาส) P1
  • %achievement → auto-score (target 8, 1–10) P1
  • แนบ evidence + Issue/Action log 1–3 P1
  • API auto-fetch actual จากระบบขาย P2

⚖️ Calibration & Defend

  • State machine (Draft→…→Finalized) P1
  • Bell Curve / N-Shape เกลี่ยมาตรฐาน P1
  • Defend/Dispute — พนักงานโต้แย้งก่อน finalize P1
  • Audit trail ทุกการปรับ P1

📈 Dashboard

  • Exec: drill-down Corporate→dept→บุคคล P1
  • Manager: calibration + สถานะหลักฐาน P1
  • Employee: Skill Radar + คำแนะนำพัฒนา P1
  • Advanced analytics / YoY P2

🛡 Governance

  • Enterprise Data Ownership (ข้อมูล = ขององค์กร) P1
  • Audit trail + RLS + เข้ารหัส · PDPA P1
  • Export ข้อมูลได้เต็ม P1
  • Granular RBAC P2

4. กด Submit
▼

Manager L1
1. รีวิว + ให้คะแนน/ปรับ → ส่งต่อ Reviewer L2 อนุมัติชั้น 2
▼

▼
Employee
3. เห็นคะแนน — ถ้าไม่ตรงหลักฐาน กด Defend แนบเหตุผล → สถานะ Pending Defend
▼
Manager/OD
4. ทบทวนพร้อมหลักฐาน → ปรับจูน → Calibrated → Finalized
Draft→Pending Calibration→Pending Defend→Calibrated→Finalized
Flow C · Skill Baseline → Gap → พัฒนา
Employee
1. ประเมิน proficiency ตนเอง (1–5) ต่อ Skill Domain + แนบ evidence
▼
Manager

▼
System
