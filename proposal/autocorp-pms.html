ขอบเขตงาน · Scope of Work
✅ In Scope (Phase 1)
  • Access: Admin login + ระบบสร้าง token link ให้พนักงาน + สิทธิ์ผู้บริหารดู dashboard
  • Import พนักงาน: นำเข้า Excel/CSV เข้า DB + ผังสายบังคับบัญชา (resource manager)
  • Master data: Master KPI, 5 แกน, กรอบ competency M/K/S, รอบประเมิน
  • KPI: กรอก target/actual + แนบหลักฐาน + คำนวณ/ตัดเกรดอัตโนมัติ
  • PM tool (ง่าย): สร้าง project + assign resource + ผูก resource↔project
  • Competency: M/K/S + sub + วัด output + แนบ evidence (link)
  • Process: Issue/Action log · workflow อนุมัติ (approver 1/2) · Calibration
  • Dashboard: สถานะการประเมิน + มุมมองผู้บริหาร
  • Deploy: ขึ้น Vercel + Supabase · ออกแบบเผื่อ Gateway สำหรับ Phase 2
⛔ Out of Scope (→ Phase 2 / แยกสัญญา)
  • เชื่อม HumanSoft SSO / API (Phase 1 ใช้ import + token link)
  • เชื่อม LMS / ERP / RT ดึง actual อัตโนมัติ
  • PM Tool แบบ Pro (task, %progress, dependency, portal)
  • Analytics ขั้นสูง / YoY / 360° Employee Data Platform
  • Migration ข้อมูลเก่าย้อนหลัง (นอกจากไฟล์ import ที่ตกลง)
  • งาน change ตาม policy หลัง sign-off (ผ่าน Change Request)
  • Hardware / license ภายนอก / ค่าบริการ cloud (ลูกค้ารับผิดชอบ)
  • Training นอกเหนือจาก handover session ที่ระบุ

การเปลี่ยนแปลงงาน · Change Request (CR)

หลัง Requirement Sign-off (M1) ความต้องการที่เพิ่ม/เปลี่ยนจากที่ตกลง ถือเป็น Change Request และคิดค่าใช้จ่ายเพิ่มตามนี้:

ขั้นตอน CR
  1. ลูกค้าแจ้งความต้องการเปลี่ยน (เป็นลายลักษณ์อักษร)
  2. Vendor ประเมิน impact (man-day + timeline)
  3. ลูกค้าอนุมัติ CR + ค่าใช้จ่าย
  4. ปรับแผน/timeline แล้วลงมือ
เกณฑ์คิดราคา CR
  • คิดที่เรตเดิม 18,000/md (ปัดขึ้นครึ่งวัน)
  • CR ที่กระทบ timeline → เลื่อนกำหนดส่งตามสัดส่วน
  • Bug (ทำงานไม่ตรง spec ที่ตกลง) = แก้ฟรีในประกัน ไม่ใช่ CR
  • Change (เพิ่ม/เปลี่ยน spec) = CR คิดเงิน

เงื่อนไข · Terms
  • การรับประกัน (Warranty): แก้ bug ในขอบเขตที่ส่งมอบ ฟรี 30 วันหลัง Go-Live
  • ทรัพย์สินทางปัญญา: โอนสิทธิ์ source code ให้ลูกค้าเมื่อชำระครบ (ยกเว้น component/library ทั่วไป)
  • การยืนราคา: 30 วันนับจากวันที่เสนอ
  • ความล่าช้าฝั่งลูกค้า: หาก dependency (ข้อมูล/การอนุมัติ/UAT) ล่าช้า กำหนดส่งเลื่อนตามสัดส่วน
  • ขอบเขตชัดเจน: งานนอก In-Scope = CR หรือ Phase 2 (แยกสัญญา)
  • * ร่างเชิงพาณิชย์เพื่อพิจารณา — เงื่อนไขทางกฎหมาย/ภาษี ควรตรวจกับฝ่ายที่เกี่ยวข้องก่อนลงนามจริง
